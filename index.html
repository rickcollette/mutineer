MUTINEER BBS

A classic telnet BBS тАФ written in C  тЦо

Sail the digital seas.

Mutineer is a full-featured telnet BBS server built from scratch in C11, styled after the classic Renegade BBS with a green pirate theme. It runs on modern Linux with SQLite persistence and supports up to 256 concurrent nodes.

From threaded message boards and offline QWK mail to real DOS doors running under DOSBox with live socket passthrough тАФ Mutineer aims for authentic BBS nostalgia with modern security and reliability.

Features

Multi-Node Telnet

Thread-per-connection architecture supports up to 256 concurrent callers. Real-time node tracking, inter-node whisper, wall broadcasts, and a WFC (Waiting for Caller) console.

Message System

Threaded message boards, private email with CC, QWK offline mail packets (with NDX index files), FidoNet netmail export, taglines, full-screen editor, and per-user scan flags.

File Areas

Upload/download with credits and ratio enforcement. Batch operations, archive test and extract (zip, tar, rar, 7z), file validation sysop command, and configurable file points.

DOS Door Runner

Runs classic DOS BBS doors under DOSBox with per-launch isolated runtime trees and serial nullmodem socket inheritance. The Aladdin Adventure door from 1992 is included as a reference.

ACS System

Full expression parser: security levels, AR/AC flags, age, time, credits, conference membership, subscription presence (E#), login counts (C#), post/call ratio (PC), download ratio (DR), and boolean operators.

Sysop Tools

Remote F-key shortcuts (F1=who, F2=broadcast, F3=kick, F4=stats, F8=twit, F10=chat). Editors for users, conferences, protocols, votes, events, and menus тАФ all accessible from within a remote session.

Chat & Paging

Split-screen two-panel ANSI chat using proper scroll regions, teleconference channel chat, sysop page with configurable attempt limit and email fallback, per-session chat logging to file.

Color Schemes

Eight named built-in color schemes selectable per-user. MCI tokens ^0тАУ^9 map through the active scheme. Sysop green pirate theme is the default.

Quick Start

Dependencies

# Debian/Ubuntu apt-get install build-essential cmake libsqlite3-dev libssl-dev dosbox # Optional: Argon2 password upgrade path apt-get install libargon2-dev

Build & Run

# Clone git clone https://github.com/rickcollette/mutineer.git cd mutineer
# Build cmake -S . -B build -DCMAKE_BUILD_TYPE=Release cmake --build build
# Initialize fresh database build/mutineer-initbbs -c conf/mutineer.conf
# Start (default port 2929) build/mutineer -c conf/mutineer.conf
# Connect telnet localhost 2929

Run Tests

cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug cmake --build build ctest --test-dir build --exclude-regex "tools_cli|expect_suite"

DOS Door Runner

Mutineer supports classic DOS BBS doors via DOSBox with full serial nullmodem socket inheritance тАФ the caller's live telnet socket is passed directly to DOSBox on launch.

PLANK Networking

PLANK (Packet Link for Area Networked Knowledge) is a store-and-forward message networking protocol for offline packet exchange between Mutineer nodes тАФ the spiritual successor to FidoNet echomail for a modern telnet BBS world.

Store & Forward

Messages are bundled into signed CBOR packets, stored locally, and exchanged with peer nodes on a configurable schedule.

Ed25519 Identity

Each node has a cryptographic identity. Bundles are signed by the originating node and verified on receipt.

plankctl / plank-offline

Admin CLI for managing links, queues, dead-letters, and quarantine. User-facing offline packet export/import tool.

Deduplication

Object-level and bundle-level deduplication prevents message storms when connected to multiple peers.

Buccaneer Scripting VM

Buccaneer is an embedded scripting language for writing interactive BBS doors. Pascal-like syntax compiles to bytecode that runs in a sandboxed stack-based VM with capability-controlled host function access.

// hello.bucc тАФ a minimal Buccaneer door PROCEDURE Main TERM.PRINTLN("Ahoy, " & USER.NAME() & "!") TERM.PRINTLN("Your security level: " & USER.SECURITY()) DOOR.EXIT END

The compiler, VM, and host bridge are in src/buccaneer/. See BUCC_TODO.md for known gaps and open implementation work.
